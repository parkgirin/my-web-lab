# Window_Alchemy

## 게임 개요

Window_Alchemy는 **뷰포트(Window)를 조작하여 원소를 상호작용시키는 퍼즐/시뮬레이션 게임**이다.

플레이어는 원소를 직접 움직이지 않는다.
대신 창의 크기와 형태, 그리고 창의 물리 법칙을 이용하여 원소를 원하는 위치로 이동시키고 새로운 원소를 만들어낸다.

---

# 핵심 시스템

## Viewport(Window)

각 창(Window)은 하나의 독립된 물리 공간이다.

창마다 서로 다른 물리 법칙을 가질 수 있다.

예시

- 중력 적용
- 중력 미적용
- 높은 압력
- 높은 온도
- 시간 정지

창의 활성화 여부에 따라 시뮬레이션이 진행된다.

---

# 원소(Element)

모든 원소는 직사각형 영역(Rectangle)을 가진다.

각 원소는 다음 속성을 가진다.

- 종류(Element Type)
- 상태(State)
- 부피(Volume)
- 질량(Mass)
- 온도(Temperature)

추가적으로 물리 속성을 가진다.

- 유체 여부
- 압축 가능 여부

---

# 뷰포트

뷰포트는 플레이어가 조작하는 대상이다.

플레이어는

- 창 크기 변경
- 창 위치 변경
- 창 활성화

등을 이용하여 게임을 진행한다.

뷰포트 변화는 실시간으로 원소의 상태에 영향을 준다.

---

# 압력

압력은

```
Pressure = Occupied Volume / Available Volume
```

개념으로 계산한다.

압력이 증가하면

- 유체가 압축될 수 있다.
- 일부 원소는 상태 변화가 발생한다.
- 특정 연금술 조합의 조건이 될 수 있다.

---

# 상태 변화

원소는 다음 조건에 의해 상태가 변할 수 있다.

- 압력
- 온도
- 다른 원소와의 접촉

예시

물 + 흙 → 점토

점토 + 불 → 벽돌

---

# 충돌

모든 원소는 직사각형 충돌 영역을 가진다.

충돌 판정은 JavaScript에서 수행한다.

CSS는 시각적 표현만 담당한다.

게임 계산 결과와 화면은 항상 동일해야 한다.

---

# 유체

유체는 자유 공간을 채우려 한다.

고체 내부에는 침투하지 않는다.

뷰포트 변화가 발생하면
유체는 새로운 공간에 맞게 다시 배치된다.

---

# 고체

고체는 자신의 형태를 유지한다.

압축되지 않는다.

다른 원소의 이동을 막는다.

---

# 연금술

특정 원소가 접촉하고 조건을 만족하면 새로운 원소가 생성된다.

예시

물 + 흙 → 점토

점토 + 불 → 벽돌

---

# 프로토타입 범위

초기 프로토타입에서는 다음만 구현한다.

원소

- 물
- 흙
- 불
- 공기
- 점토
- 벽돌

물리

- 직사각형 충돌
- 유체 공간 채우기
- 압력 계산

연금술

물 + 흙 → 점토

점토 + 불 → 벽돌

창

- 중력 ON
- 중력 OFF

두 종류만 구현한다.

---

# 구현 원칙

CSS는 화면을 표현한다.

JavaScript는 게임 상태를 계산한다.

화면과 계산 결과는 항상 동일해야 한다.

게임의 규칙은 CSS에 의존하지 않는다.
CSS는 계산 결과를 표현하는 역할만 수행한다.
